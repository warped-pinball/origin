Component guide

Component library patterns are listed below. Use these HTML + CSS patterns to assemble pages quickly without a frontend framework. Each example includes markup that can drop straight into Jinja macros.

Responsive by default

Page shells

Standard shells, gutters, and backgrounds keep admin screens aligned with public pages.

  • Shells

    page-shell (static pages) · c-shell (component demos)

    Both shells share --page-max-width (1200px) and --page-gutter (24px) spacing.

  • Backgrounds

    --page-background for spiral skies · --page-background-plain for focused tools

    Apply page--plain on admin detail pages like /admin/machines/<id> to keep the surface white.

  • Space scale

    --space-050–--space-1200

    Use the shared spacing variables for gaps and padding to keep rhythm between sections.

Usage All components rely on the c- namespace and fluid grids to prevent overflow.

  • Drop-in macros

    Wrap card content with c-card and lists with c-list.

    Macros keep markup consistent and let you swap data easily between debug and live views.

  • Responsive grid

    Use minmax(0, 1fr) columns

    List rows and cards stretch to the available space while avoiding horizontal scroll.

  • Overflow safety

    Text wraps and score values stay readable

    Long names, dates, and scores are clamped with flex + grid alignment.

Navigation (c-nav) Wrap primary links; items wrap automatically on smaller screens.
